f:Tabe,<h2>🚨WARN : Possible Unhandled Promise Rejection</h2>
<h2>Error: [storage/unauthorized] User is not authorized to perform the desired action.</h2>
<p>Firebase Storage를 사용해서 처음 이미지를 업로드 하려는데!<br>
경고와 함께 무한 로딩이 떴다.<br>
<code>WARN : Possible Unhandled Promise Rejection</code></p>
<p>봐도 무슨 경고인지 모르겠던 나는 아마도 정상.
이미지 업로드를 하는 부분에서 에러를 처리해주지 않아서 뭐가 문제인지 에러가 안나오는거다.</p>
<p>memo... async await로 비동기처리를 한 함수들은 꼭 에러처리도 잘 해주자...✏️</p>
<pre><code>// try catch를 이용해서 에러 발생시 콘솔에 출력해주기.
try {
  if (Platform.OS === 'android') {
    await reference.putString(asset.base64, 'base64', {
      contentType: asset.type,
    });
  } else {
    await reference.putFile(asset.uri);
  }
} catch (err) {
  console.error(err);
}
</code></pre>
<p>그러면 이제 진짜 에러가 모습을 드러낸다.</p>
<p><code>Error: [storage/unauthorized] User is not authorized to perform the desired action.</code></p>
<p>이 에러는 Firebase Storage의 권한 설정 문제이다.</p>
<p>Firebase 프로젝트 콘솔 웹페이지에 접속해서 빌드 > Storage 페이지에 가면, Rules 탭이 있다.<br>
거기에서 데이터를 읽고 쓰는 권한을 설정할 수 있다.</p>
<pre><code>rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if false;
    }
  }
}
</code></pre>
<p>read, write 권한이 <code>false</code>로 되어있어서 오류가 났다.</p>
<pre><code>rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write;
    }
  }
}
</code></pre>
<p>위와 같이 <code>allow read, write</code> 라고 작성하면 읽기 쓰기 권한 모두 <code>true</code>로 허용된다.
설정을 마치고 적용되기를 잠시 기다리면 잘 작동한다!</p>
<p>모닝에러 클리어👍</p>
